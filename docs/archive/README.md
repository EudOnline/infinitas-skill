# Archive

Archive pages preserve history, plans, and superseded narratives. They are useful for context, but they should not be treated as the default source of current operating truth.

## What lives here conceptually

- implementation plans and design drafts
- closeout reports and historical milestones
- old audits, exploratory writeups, and one-off research documents

## Current archive entrypoints

- [Maintainability and docs reset roadmap](../plans/2026-03-30-maintainability-and-docs-reset-roadmap.md)
- Historical implementation plans under `docs/plans/`
- [Design and UX explorations](design/ui-ux-analysis-and-rebuild.md)
- [Theme migration history](design/theme-migration-guide.md)
- [Audit snapshots](audits/dark-mode-audit.md)
- [Project closeout](../project-closeout.md)

## How to use archive material

- Use archive docs to understand why a change happened.
- Use guide, reference, ops, and ADR pages to understand what is canonical now.
- When an archived page still contains useful truth, promote that truth into a maintained page instead of asking readers to memorize old plan documents.
