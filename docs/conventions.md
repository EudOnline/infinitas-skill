# Repository Conventions

## Skill layout

Each skill should usually follow this pattern:

```text
skill-name/
├─ SKILL.md
├─ scripts/
├─ references/
└─ assets/
```

## Naming

- Use lowercase letters, digits, and hyphens
- Keep names short and specific
- One skill = one folder

## Authoring rules

- `SKILL.md` is required
- Put reusable executable helpers in `scripts/`
- Put reference docs in `references/`
- Put output resources/templates in `assets/`
- Keep secrets out of git
